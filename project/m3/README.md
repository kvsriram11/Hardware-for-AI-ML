## File Catalog

### RTL

| File | Description |
|------|-------------|
| `rtl/top.sv` | M3 top-level module. Thin wrapper that instantiates `interface_axi` (M2). No glue logic. All AXI4-Lite + AXI4-Stream ports exposed. Includes `ifdef COCOTB_SIM` VCD dump block (compile-time guarded, never seen by synthesis). |

### Testbench

| File | Description |
|------|-------------|
| `tb/Makefile` | cocotb 2.0 build system. Sets `SIM=icarus`, `TOPLEVEL=top`, `MODULE=test_top`, lists RTL sources, sets `WAVES=1` for FST waveform capture, includes `Makefile.sim` (not deprecated `Makefile.inc`). |
| `tb/test_top.py` | cocotb 2.0.1 end-to-end co-simulation testbench. Drives `top` exclusively through AXI4-Lite (config writes + result reads) and AXI4-Stream (N=64 operand beats). Includes Q15 golden model. Prints `PASS` or `FAIL` with numeric comparison. |

### Simulation Artifacts

| File | Description |
|------|-------------|
| `sim/cosim_run.log` | Full `make` output from the passing co-simulation run. Confirms: Icarus 12.0 compile (5 expected "sorry" warnings), cocotb 2.0.1 test runner, and `PASS | cosim_end_to_end | dut=-2022 golden=-2022` at 1550 ns. |
| `sim/cosim_waveform.png` | Matplotlib waveform showing three annotated regions: Region A (0–180 ns, AXI4-Lite writes), Region B (180–1465 ns, AXI4-Stream + internal compute), Region C (1465–1550 ns, AXI4-Lite reads). Generated by `sim/plot_waveform.py` from FST→VCD conversion of `tb/sim_build/top.fst`. |
| `sim/plot_waveform.py` | Python script that parses `dump.vcd` and renders `cosim_waveform.png` using matplotlib. Usage: `python plot_waveform.py [vcd_path] [output_png]`. |

## Reproduce Co-Simulation

```bash
cd project/m3/tb
make clean
make
# Output: PASS | cosim_end_to_end | dut=-2022 golden=-2022
# Waveform: sim_build/top.fst
```

To regenerate cosim_waveform.png:
```bash
# Convert FST to VCD (requires GTKWave)
fst2vcd tb/sim_build/top.fst > tb/sim_build/dump.vcd
# Render PNG
cd sim
python plot_waveform.py ../tb/sim_build/dump.vcd cosim_waveform.png
```

## Reproduce Synthesis

```bash
# Requires Yosys 0.52+ in PATH (WSL Ubuntu: sudo apt install yosys)
# Requires sky130A PDK at ~/.volare/ (from prior OpenLane 2 install)

cd project/m3/synth

# Generic synthesis (area stats, no cell library)
yosys -l openlane_run.log synth_top.ys

# Technology-mapped synthesis — area in um2 (fast, <4s)
yosys -l synth_area.log synth_area.ys

# Technology-mapped synthesis with timing info (print_stats per module)
yosys -l synth_timing.log synth_timing.ys
```

To attempt full OpenLane 2 flow (requires Docker with EDA tools):
```bash
# Start Docker Desktop, then:
source ~/openlane2_work/venv/bin/activate
cd ~/openlane2_work/designs/top
openlane config.json
```

## Module Hierarchy

```
top  [project/m3/rtl/top.sv]
└── interface_axi  [project/m2/rtl/interface.sv]
    └── compute_core  [project/m2/rtl/compute_core.sv]
        ├── q15_mac    [project/m2/rtl/q15_mac.sv]
        ├── q15_tanh   [project/m2/rtl/q15_tanh.sv]
        └── q15_blend  [project/m2/rtl/q15_blend.sv]
```

The testbench drives `top` exclusively through AXI4-Lite (register file access)
and AXI4-Stream (operand streaming). No internal signals of `interface_axi` or
`compute_core` are accessed directly by the testbench. This satisfies the M3
co-simulation isolation rule.
