# M2 — Compute Core, Interface, and Verification

## Overview

This milestone delivers the synthesizable RTL for the ESN-accelerator chiplet's compute core and 
its host-side AXI interface, with a cocotb-based regression that compares against a bit-exact 
Python reference and produces PASS/FAIL logs and an annotated waveform.

## What's in this directory

- `mac_array.sv` — 16-wide parallel signed MAC array, pipelined (mul → reduce → accumulate)
- `tanh_pwl.sv` — 4-segment piecewise-linear tanh approximation
- `leak_blend.sv` — `(1-a)x + a*z` blender with Q1.15 truncation
- `compute_core.sv` — top-level state-update FSM that orchestrates MAC → tanh → blend
- `interface.sv` — AXI4-Lite slave (control/status) + AXI4-Stream sink (W, x chunks)
- `precision.md` — Q15 format rationale + 200-sample quantization error analysis
- `quantization_stats.json` — raw numerical results
- `build_m2_docs.py` — regenerates this README and precision.md from current data

Testbench, simulation logs, and waveforms live at:

- `tb/m2/golden.py` — bit-exact Python reference
- `tb/m2/test_compute_core.py` — 3 cocotb tests for the compute core
- `tb/m2/test_interface.py` — 2 cocotb tests for the AXI wrapper
- `tb/m2/runner.py` — cocotb 2.0 runner entry point
- `tb/m2/vcd_to_png.py` — VCD → annotated waveform renderer
- `sim/compute_core_run.log` — cocotb regression log, contains PASS lines
- `sim/interface_run.log` — cocotb regression log, contains PASS lines
- `sim/waveform.png` — annotated waveform of `test_representative_vector`
- `sim/waveform_gtkwave.png` — supplementary GTKWave capture of the same simulation

## Reproduce the simulation

From a fresh clone, in Git Bash:

```bash
cd ESN-redo
source env/venv/Scripts/activate
cd tb/m2
python -u runner.py compute_core --waves 2>&1 | tee ../../sim/compute_core_run.log
python -u runner.py interface    --waves 2>&1 | tee ../../sim/interface_run.log
python vcd_to_png.py             # regenerates sim/waveform.png
```

Expected results:

- `compute_core`: `TESTS=3 PASS=3 FAIL=0 SKIP=0`
- `interface`:    `TESTS=2 PASS=2 FAIL=0 SKIP=0`
- `sim/waveform.png` and `sim/waveform_gtkwave.png` both present, both > 50 KB

### VCD note

Cocotb 2.0's Icarus runner emits FST waveforms by default when `--waves` is set. 
To convert to VCD for tools that expect VCD specifically, run:

```bash
fst2vcd -f sim_build/<top>_DW16/waves.fst -o sim_build/<top>_DW16/waves.vcd
```

`fst2vcd` ships with GTKWave and is at `/c/iverilog/gtkwave/bin/` on this Windows install.

## Multi-precision parameterization

All RTL is parameterized by `DATA_W` (defaulting to 16 for Q15). To re-run the same testbench at INT8 or Q4:

```bash
python runner.py compute_core --data-w 8   --waves   # INT8
python runner.py compute_core --data-w 4   --waves   # Q4
```

M2 ships Q15 as the reference precision. INT8 and Q4 are characterized in M4 
(see `project/m4/bench/` once M4 is complete).

## Deviations from M1

None substantive. 
The M1 partition placed compute (MAC + tanh + leak) on the chiplet, with the host running spectral normalization and readout training; M2 implements exactly that compute core. 
The M1 interface selection chose AXI4-Stream (data) + AXI4-Lite (control); M2's `interface.sv` implements both per AXI spec section A2. 
The M1 system diagram showed on-chip SRAM for W, Win, and x; M2's compute core is the datapath fed by that SRAM, 
but the SRAM and weight-loading sequencer are implemented at the M3 top-level integration, 
not in this M2 deliverable. 
M2 verifies the compute path on a single-chunk, single-neuron representative vector, which is the rubric's required scope.

## Architectural notes

- Single clock domain (`clk`), active-low synchronous reset (`rst_n`).
- 6-state FSM: IDLE → MAC → FLUSH → ACTIVATE → BLEND → DONE → IDLE
- Pipeline latency per neuron: 9 cycles (3-stage MAC pipeline + 3 flush + 1 activate + 1 blend + 1 done)
- Target clock: 100 MHz on sky130; expected to close timing comfortably given the pipelined module boundaries
- AXI register map (AXI4-Lite, byte addressed, 32-bit data):
  - `0x00 CTRL` — bit[0]=start, bit[1]=soft_rst, bit[2]=mode_run
  - `0x04 STATUS` — bit[0]=busy, bit[1]=done, bit[2]=err
  - `0x08 LEAK_A` — signed Q1.(DATA_W-1) leak coefficient
  - `0x0C WIN_T` — low-32 of the signed ACC_W Win-projection precomputed scalar
  - `0x10 XPREV` — signed DATA_W prior state for the current neuron
  - `0x14 XNEXT` — read-only DATA_W computed next state
